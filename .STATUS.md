# gRPC Debugger 狀態報告 (v1.0.0)

## 📌 專案概況
本專案已完成從 React 到 **Svelte 5 (Runes)** 的全新架構遷移。目前專案處於穩定狀態，支援 **gRPC-Web** 與 **Connect-RPC** 的自動偵測與解碼。

## 🛠️ 技術棧
- **Frontend**: Svelte 5 (Runes 模式)
- **Build Tool**: Vite + ESM Standard
- **Decoding Engine**: 自研 `ProtoEngine.js` (Schema-Driven)
- **Communication**: Manifest V3 + Chrome Debugger API

## ✅ 已完成功能
- [x] **自動 Reflection**: 偵測到新伺服器時，背景會自動嘗試獲取定義。
- [x] **定義驅動解碼**: 優先使用 Protobuf 定義還原欄位名稱與巢狀結構。
- [x] **盲測解碼 (Blind Decoding)**: 在缺定義的情況下，透過啟發式演算法還原數據層級。
- [x] **多頁面導航**: 網路紀錄 (Network)、服務定義 (Services)、設定 (Settings) 頁面。
- [x] **完整國際化 (i18n)**: 支援 繁體中文 (zh) 與 英文 (en)。
- [x] **舊程式碼清理**: 已清除所有舊版 React 檔案。

## ✅ 最近修復 (Last Commit)
1. **重構解析引擎**: `ProtoReader` 全面改用 `BigInt` 與 `DataView`，徹底消除 32 位元溢出。
2. **傳輸層透明化**: 支援 `gzip` 自動解壓縮與 Connect-RPC 影格對齊（過濾 EOS 影格）。
3. **反射定義加強**: 補全 Enum 定義解析與 Nested Types 完整路徑校正。

## 📋 殘留問題：無
目前已達成 Schema-Driven 的精確解析。
